# 로컬 실행 가이드

## 메타
- 문서 목적: standalone package를 로컬 PC에서 직접 실행하고, 대시보드나 HTML 리포트가 `127.0.0.1:3010`으로 AI 호출을 수행하도록 설정합니다.
- 대상 독자: 개인 워크스테이션 운영자, 로컬 대시보드 개발자, HTML 리포트 사용자
- 기준 운영안: `kits/myagent-copilot-kit/standalone-package`
- 현재 기준 버전/산출물: `v0.1.0`, `release/myagent-copilot-standalone-v0.1.0.zip`
- 관련 문서:
  - `02-사전준비-체크리스트.md`
  - `07-검증-체크리스트.md`
  - `../standalone-package/README.md`

## 1. 목표 상태
이 문서를 끝내면 다음 상태가 되어야 합니다.
- 로컬 프록시가 `http://127.0.0.1:3010`에서 실행된다.
- `GET /api/ai/health`가 정상 응답한다.
- `POST /api/ai/chat`에 프롬프트를 보내면 응답이 온다.
- 프런트엔드 또는 HTML 리포트에서 AI 기능이 동작한다.

## 2. Windows 절차

### 2.1 폴더 이동
```powershell
cd C:\Users\jichu\Downloads\openclaw\kits\myagent-copilot-kit\standalone-package
```

### 2.2 의존성 설치
```powershell
pnpm install
```

### 2.3 빌드
```powershell
pnpm build
```

### 2.4 GitHub 로그인
```powershell
pnpm login
```

정상이라면 터미널에 다음 형태의 안내가 나옵니다.
- `Open: https://github.com/login/device`
- `Code: XXXX-XXXX`
- 승인 후 `~/.myagent-copilot/auth-profiles.json`에 토큰 저장

### 2.5 로컬 서버 실행
둘 중 하나를 씁니다.

직접 명령:
```powershell
pnpm serve:local
```

배치 파일:
```powershell
.\serve-local.bat
```

### 2.6 헬스체크
```powershell
Invoke-RestMethod http://127.0.0.1:3010/api/ai/health
```

기대값 예시:
```json
{
  "ok": true,
  "service": "myagent-copilot-standalone",
  "host": "127.0.0.1",
  "port": 3010
}
```

## 4. AI 호출 테스트

### 4.1 최소 POST 예시
```bash
curl -X POST http://127.0.0.1:3010/api/ai/chat \
  -H "Content-Type: application/json" \
  -H "x-request-id: local-smoke-001" \
  -d '{
    "model": "github-copilot/gpt-5-mini",
    "sensitivity": "internal",
    "messages": [
      { "role": "system", "content": "입력 JSON 바깥 사실은 단정하지 말라." },
      { "role": "user", "content": "테스트 응답을 2문장으로 반환하라." }
    ]
  }'
```

### 4.2 기대 응답 구조
```json
{
  "requestId": "local-smoke-001",
  "route": "copilot",
  "sensitivity": "internal",
  "result": {
    "text": "...",
    "model": "gpt-5-mini",
    "provider": "github-copilot",
    "endpoint": "responses"
  },
  "guard": {
    "dlpStatus": "allow",
    "reason": "Request is eligible for Copilot route."
  }
}
```

## 5. HTML 리포트 또는 대시보드 연결

### 5.1 런타임 주입 예시
```html
<script>
  window.__JPT71_AI_ENDPOINT__ = "http://127.0.0.1:3010/api/ai/chat";
</script>
```

### 5.2 fetch 예시
```ts
const response = await fetch("http://127.0.0.1:3010/api/ai/chat", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-request-id": crypto.randomUUID(),
    "x-ai-sensitivity": "internal",
  },
  body: JSON.stringify({
    model: "github-copilot/gpt-5-mini",
    sensitivity: "internal",
    messages: [
      {
        role: "system",
        content: "입력 JSON에 포함된 수치와 필터만 사용하라.",
      },
      {
        role: "user",
        content: "현재 선택된 필터에서 핵심 원인 3개를 요약하라.",
      },
    ],
  }),
});
```

## 6. 로컬에서 자주 보는 문제

### 6.1 `AI proxy unavailable`
- 프록시 프로세스가 꺼졌는지 확인합니다.
- `GET /api/ai/health`가 되는지 먼저 확인합니다.

### 6.2 `DLP requires sanitization and policy forbids sending sanitized payloads to Copilot`
- 메시지에 이메일, 전화번호, credential-like token이 섞였을 가능성이 큽니다.
- payload를 줄이거나, 민감 요소를 제거하거나, local-only 경로를 구성해야 합니다.

### 6.3 `PROXY_AUTH_REQUIRED`
- 로컬 모드에서는 보통 토큰을 안 쓰지만, `MYAGENT_PROXY_AUTH_TOKEN`을 설정해 둔 상태일 수 있습니다.

## 7. 종료 방법
- 실행 중인 터미널에서 `Ctrl+C`
- 또는 별도 콘솔 프로세스를 종료

### 주의사항
- 로컬 모드에서 동작한다고 해서 퍼블릭 배포에서도 그대로 동작하는 것은 아닙니다. 퍼블릭에서는 CORS, HTTPS, 프록시 토큰 인증을 추가로 검증해야 합니다.
- JPT71 같은 리포트에는 원본 `DATA` 전체를 넣지 말고 필터/집계 요약만 보내십시오.
