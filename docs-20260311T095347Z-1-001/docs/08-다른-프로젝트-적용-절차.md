# 다른 프로젝트 적용 절차

## 메타
- 문서 목적: 현재 키트를 JPT71 외 다른 HTML 리포트, React 대시보드, 내부 분석 화면에 재사용하는 방법을 설명합니다.
- 대상 독자: 프런트엔드 개발자, 대시보드 제작자, 신규 프로젝트 담당자
- 기준 운영안: `kits/myagent-copilot-kit/standalone-package`
- 현재 기준 버전/산출물: `v0.1.0`, `release/myagent-copilot-standalone-v0.1.0.zip`
- 관련 문서:
  - `03-로컬-실행-가이드.md`
  - `04-퍼블릭-배포-가이드.md`
  - `05-보안-운영-가이드.md`
  - `../../MYAGENT-PROXY-FRONTEND-AND-LOG-SCHEMA.md`

## 1. 재사용 원칙
- 원본 데이터 전체를 보내지 말고, 필터/집계 요약만 보냅니다.
- 엔드포인트는 항상 `GET /api/ai/health`, `POST /api/ai/chat`만 사용합니다.
- 기본 모델은 `github-copilot/gpt-5-mini`를 유지합니다.
- 공개 배포라면 `MYAGENT_PROXY_AUTH_TOKEN`, `MYAGENT_PROXY_CORS_ORIGINS`를 먼저 정합니다.

## 2. 적용 순서
1. 대시보드에서 AI UI를 넣을 위치를 정합니다.
2. 현재 화면 상태에서 필요한 최소 요약 JSON 스키마를 정의합니다.
3. 프롬프트 템플릿을 3~4개 수준으로 고정합니다.
4. 프런트에서 `/api/ai/chat` 호출 코드를 넣습니다.
5. 로컬에서 먼저 검증합니다.
6. 퍼블릭이 필요하면 엔드포인트, 토큰, CORS를 전환합니다.

## 3. 무엇을 보내고 무엇을 보내지 않을 것인가

### 3.1 보내도 되는 것
- 현재 필터 값
- KPI 집계값
- 상위 이슈 5~10개
- 비교 대상 A/B 핵심 수치
- 데이터 품질 점수

### 3.2 보내지 말 것
- 전체 raw dataset
- CSV 전체
- 고객 실명, 전화번호, 이메일
- access token, key, credential
- 내부 메모 전체 덤프

## 4. UI 패턴 예시
- 프리셋 버튼 3개
  - 원인 요약
  - 실행 액션
  - 리스크 점검
- 자유 질문 textarea
- 응답 영역
- 로딩/오류 박스

## 5. 프런트 샘플

### 5.1 fetch
```ts
async function askAi(summaryPayload: unknown, userQuestion: string) {
  const response = await fetch(window.__AI_ENDPOINT__, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-request-id": crypto.randomUUID(),
      "x-ai-sensitivity": "internal",
      ...(window.__AI_PROXY_TOKEN__
        ? { "x-ai-proxy-token": window.__AI_PROXY_TOKEN__ }
        : {}),
    },
    body: JSON.stringify({
      model: "github-copilot/gpt-5-mini",
      sensitivity: "internal",
      messages: [
        {
          role: "system",
          content: "주어진 JSON 밖 사실은 단정하지 말고, 숫자와 단위를 유지하라.",
        },
        {
          role: "user",
          content: `질문: ${userQuestion}\n요약 JSON:\n${JSON.stringify(summaryPayload)}`,
        },
      ],
    }),
  });
  return response.json();
}
```

### 5.2 axios
```ts
import axios from "axios";

const client = axios.create({
  baseURL: window.__AI_ENDPOINT_BASE__,
});
```

세부 예시는 `../examples/frontend-axios.ts`를 그대로 사용하면 됩니다.

## 6. JPT71 패턴을 다른 프로젝트로 옮길 때
- AI 패널은 기존 탭 흐름을 깨지 않는 위치에 둡니다.
- 요약 JSON 빌더 함수를 하나 둡니다.
- 프리셋 프롬프트는 운영 목적별로 고정합니다.
- AI 응답은 HTML 렌더링이 아니라 plain text로 출력합니다.

## 7. 로컬과 퍼블릭의 분기

### 7.1 로컬
- 엔드포인트: `http://127.0.0.1:3010/api/ai/chat`
- 토큰: 선택
- CORS: 로컬 origin

### 7.2 퍼블릭
- 엔드포인트: `https://api.your-domain.com/api/ai/chat`
- 토큰: 필수
- CORS: 실제 서비스 origin만 허용

## 운영 공통 블록
