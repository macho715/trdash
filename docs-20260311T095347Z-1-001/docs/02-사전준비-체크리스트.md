## 1. 런타임 준비

### 1.1 Node
- 요구 버전: Node `22.12.0` 이상
- 확인 명령:
```bash
node -v
```

### 1.2 pnpm
- 요구: `pnpm` 설치 완료
- 확인 명령:
```bash
pnpm -v
```

### 1.3 압축 해제 또는 소스 복사
둘 중 하나를 선택합니다.
- 저장소에서 `kits/myagent-copilot-kit/standalone-package` 폴더를 그대로 복사
- `release/myagent-copilot-standalone-v0.1.0.zip`를 해제

## 2. 인증 준비

### 2.1 기본 인증 경로
가장 권장하는 순서는 아래와 같습니다.
1. `pnpm login`으로 standalone store에 직접 로그인
2. 환경변수 `MYAGENT_GITHUB_TOKEN` 또는 `GH_TOKEN` 사용
3. 기존 OpenClaw auth store fallback 사용

### 2.2 상태 파일 경로 확인
- standalone auth store:
  - `~/.myagent-copilot/auth-profiles.json`
- runtime token cache:
  - `~/.myagent-copilot/cache/github-copilot.token.json`
- OpenClaw fallback auth:
  - `~/.openclaw/agents/main/agent/auth-profiles.json`
  - `~/.openclaw/auth-profiles.json`

### 2.3 파일 권한
- 가능하면 현재 사용자만 읽을 수 있게 제한합니다.
- 특히 공유 서버에서는 홈 디렉터리 권한을 먼저 확인합니다.

## 3. 포트와 네트워크

### 3.1 로컬 기본 포트
- `3010`

### 3.2 충돌 확인
```bash
netstat -ano | findstr 3010
```
또는
```bash
ss -ltnp | grep 3010
```

### 3.3 퍼블릭 배포 시 추가 확인
- 외부에서 접근 가능한 서버인지
- HTTPS termination을 어디서 할지
- 리버스 프록시가 있는지
- `MYAGENT_PROXY_CORS_ORIGINS`에 실제 대시보드 origin을 넣었는지

## 4. 프런트엔드 준비

### 4.1 로컬 대시보드
- 기본 엔드포인트: `http://127.0.0.1:3010/api/ai/chat`
- 기본 허용 origin:
  - `http://127.0.0.1:4173`
  - `http://127.0.0.1:18789`

### 4.2 퍼블릭 대시보드
- 엔드포인트를 `https://api.your-domain.com/api/ai/chat` 형태로 바꿉니다.
- CORS allowlist에 Vercel 도메인을 넣습니다.
- 토큰 인증을 반드시 켭니다.

## 5. 문서/운영 기준값 사전 확인
설치 전에 아래 표현을 팀 내부에서 합의해 두는 것이 좋습니다.
- 기본 모델: `github-copilot/gpt-5-mini`
- 기본 엔드포인트: `GET /api/ai/health`, `POST /api/ai/chat`
- 기본 운영안: `Standalone 우선`
- OpenClaw 표현: `OpenClaw 경유는 호환 경로`
- 사용량 표현: `무차감 관측`, `정책상 무제한 확정 불가`

## 운영 공통 블록
### 검증 명령
```bash
cd ../standalone-package
pnpm install
pnpm build
pnpm health
```

### 주의사항
- `pnpm build` 이전에 Node 버전이 낮으면 타입스크립트 빌드가 통과하지 않을 수 있습니다.
- OpenClaw auth store fallback이 존재하더라도, 새 운영 환경에서는 standalone login을 별도로 수행하는 편이 관리가 단순합니다.
