## 1. 기본 보안 원칙
- `Standalone 우선`
- `OpenClaw 경유는 호환 경로`
- 프런트는 최소한의 요약 JSON만 전송
- 고정 장기 토큰을 브라우저에 직접 두지 않는 방향으로 개선
- `secret` 이상 민감도는 기본적으로 local-only
- `무차감 관측`과 `정책상 무제한 확정 불가`를 혼동하지 않음

## 2. DLP 기본 정책
현재 기본값은 다음과 같습니다.
- `sanitizeAtOrAbove: medium`
- `blockAtOrAbove: high`

### 2.1 medium 예시
- 이메일
- 전화번호

이 경우:
- sanitize 발생 가능
- `MYAGENT_PROXY_ALLOW_SANITIZED_TO_COPILOT=0`이면 Copilot route 금지

### 2.2 high/critical 예시
- `Bearer ...`
- GitHub PAT
- private key block
- password/token assignment
- OpenAI style key

이 경우:
- 기본적으로 block

## 3. 민감도 라우팅
민감도는 다음 우선순위로 결정됩니다.
1. request body의 `sensitivity`
2. `x-ai-sensitivity` header
3. 기본값 `internal`

기본 정책:
- `defaultSensitivity: internal`
- `minSensitivityForLocalOnly: secret`

따라서:
- `public`, `internal`, `confidential`은 DLP 조건을 만족하면 Copilot route 가능
- `secret`는 local route 강제

## 4. sanitize payload 정책

### 4.1 기본 권장값
- `MYAGENT_PROXY_ALLOW_SANITIZED_TO_COPILOT=0`

이유:
- sanitize가 필요하다는 것은 이미 민감 정보가 payload에 섞였다는 뜻입니다.
- 운영자가 payload를 더 줄이거나, local-only로 전환하는 편이 안전합니다.

### 4.2 예외적으로 `1`을 줄 수 있는 경우
- 완전히 내부망
- 전송 데이터가 집계 JSON 수준
- 민감도 분류와 DLP 결과를 운영 로그로 남김
- 보안 책임자가 승인한 경우

## 5. 토큰 운영

### 5.1 저장 위치
- auth store: `~/.myagent-copilot/auth-profiles.json`
- runtime token cache: `~/.myagent-copilot/cache/github-copilot.token.json`

### 5.2 fallback
- `~/.openclaw/agents/main/agent/auth-profiles.json`
- `~/.openclaw/auth-profiles.json`

### 5.3 권장 정책
- 장기 운영 서버는 standalone auth store로 별도 로그인
- OpenClaw fallback은 이관 단계에서만 사용
- 공유 서버에서는 홈 디렉터리 ACL을 별도로 점검

## 6. CORS와 프록시 토큰

### 6.1 CORS
- `MYAGENT_PROXY_CORS_ORIGINS`에 실제 대시보드 origin만 넣습니다.
- 와일드카드 허용은 지양합니다.

### 6.2 프록시 토큰
- 공개 배포에서는 `MYAGENT_PROXY_AUTH_TOKEN` 필수
- 프런트는 `x-ai-proxy-token` 또는 `Authorization: Bearer ...`로 전달 가능
- 토큰 노출이 우려되면 브라우저 직접 호출 대신 서버사이드 BFF 계층을 둡니다.

## 7. 운영 로그
기본 스키마는 `openclaw.copilot.proxy.log.v1`입니다.

주요 필드:
- `requestId`
- `stage`
- `outcome`
- `route`
- `httpStatus`
- `latencyMs`
- `model`
- `sensitivity`
- `dlpStatus`
- `dlpHighestSeverity`
- `sanitized`
- `usage`

로그 원칙:
- 원문 payload 전체를 남기지 않음
- requestId 중심으로 추적
- 에러 상세는 320자 이내로 절삭

## 8. 비용/사용량 표현
문서와 대시보드 문구는 아래 기준을 지킵니다.
- 실측상 quota delta가 없으면 `무차감 관측`
- GitHub 제품 정책 전체에 대한 단정은 `정책상 무제한 확정 불가`

이 구분은 감사 대응과 향후 정책 변경 대응에 중요합니다.

## 9. 금지 패턴
- 프런트에서 원본 데이터셋 전체 전송
- 공개 프록시에 `MYAGENT_PROXY_AUTH_TOKEN` 미설정
- `0.0.0.0` 공개 + CORS 광범위 허용
- 운영 로그에 원문 비밀값 출력
- OpenClaw fallback만 믿고 standalone auth store를 영구적으로 비워 두는 방식
